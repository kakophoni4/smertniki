# Rusprofile Monitor — Telegram-бот мониторинга лавок

Приватный бот проверяет компании по **достоверности сведений в ЕГРЮЛ (ФНС)**.

Источник правды: `egrul.nalog.ru` (поиск + официальная выписка PDF).  
Rusprofile не парсим (антибот/403 с VPS) — только ссылка «посмотреть глазами» в алертах.

## Что проверяется

| Сигнал | Откуда |
|--------|--------|
| Недостоверность адреса | выписка ЕГРЮЛ, блок адреса → «сведения недостоверны» |
| Недостоверность ДЛ | выписка, блок лица без доверенности |
| Недостоверность учредителя | выписка, блок учредителей |
| Предстоящее исключение / ликвидация | выписка, «Сведения о состоянии» |

## Быстрый старт (Docker)

```bash
cp .env.example .env
# BOT_TOKEN, ADMIN_IDS

docker compose up -d --build
docker compose logs -f
```

## Кнопки

После `/start` — меню.

- 🏪 Лавки → добавить / список / файл / удалить — всё по **ИНН**
- 🔍 Проверка → все или одна по **ИНН**
- 🎫 Тикеты → «Вылечить»
- 👥 Пользователи

ОГРН бот достаёт сам из ЕГРЮЛ и хранит внутри для выписки.

## Скорость и хранение

На компанию: поиск + заказ выписки + PDF (~5–15 сек).  
~200 лавок ≈ 20–40 мин. Расписание по умолчанию 2 раза в день.

Выписки: `data/vypiski/{ОГРН}.pdf` — **одна на компанию**, при новой проверке старая перезаписывается.  
В БД хранятся только последние `KEEP_CHECK_RESULTS` (по умолчанию 5) результатов проверки на лавку.

## Структура

```
app/services/egrul.py              — ФНС: поиск + выписка + парсер
app/services/rusprofile_client.py  — фасад (внутри ЕГРЮЛ)
app/bot/                           — Telegram UI
```
